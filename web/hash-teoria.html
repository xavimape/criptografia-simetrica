01 / 08

Que es una funcion hash criptografica?

Una funcion hash criptografica toma una entrada de longitud arbitraria y produce una salida de longitud fija llamada digest o resumen.

Tres propiedades fundamentales de seguridad:

  • Resistencia a preimagen: dado H, no es factible encontrar M tal que hash(M) = H.
  • Resistencia a segunda preimagen: dado M1, no es factible encontrar M2 ≠ M1 con el mismo hash.
  • Resistencia a colisiones: no es factible encontrar M1 ≠ M2 tal que hash(M1) = hash(M2).
Mensaje (longitud variable) "hello world" → b94d27b9... (SHA-256, 256 bits) "Hello world" → a4f17b20... (SHA-256, 256 bits) archivo de 4 GB → 3e23e816... (SHA-256, 256 bits) ↕ 1 bit de diferencia → ~50% de bits del digest distintos (efecto avalancha)
02 / 08

Construccion Merkle-Damgard

La mayoria de las funciones hash clasicas (MD5, SHA-1, SHA-2) usan la construccion iterativa Merkle-Damgard: se divide el mensaje en bloques de tamano fijo, se aplica una funcion de compresion a cada bloque junto con el estado anterior, y el resultado final es el hash.

Mensaje + padding ┌──────┬──────┬──────┬──────┐ │ B0 │ B1 │ B2 │ B3 │ bloques de 512 bits └──────┴──────┴──────┴──────┘ │ │ │ │ H0─┤ H1──┤ H2──┤ H3──┤ f() f() f() f() └── digest final

Vulnerabilidad inherente: la construccion Merkle-Damgard es susceptible a ataques de extension de longitud (length extension attack). SHA-3 y HMAC resuelven este problema.

03 / 08

MD5 — roto pero ubicuo

MD5 (1991) produce un digest de 128 bits (32 caracteres hex). Opera sobre bloques de 512 bits con un estado de cuatro palabras de 32 bits (A, B, C, D). Aplica 4 rondas de 16 operaciones cada una usando funciones logicas F, G, H, I.

  • 2004: Wang & Yu demuestran colisiones analiticas.
  • 2012: malware Flame usa colisiones MD5 para falsificar certificados de Microsoft.
  • Hoy: colisiones en segundos en hardware comun. No usar para seguridad.

Aun aparece en: checksums de descargas no criticas, sistemas legado, y como identificador de ficheros en plataformas de threat intelligence (MD5 de malware en VirusTotal).

Ejemplo de colision MD5: file1.bin → 79054025255fb1a26e4bc422aef54eb4 (identico!) file2.bin → 79054025255fb1a26e4bc422aef54eb4 (contenido diferente)
04 / 08

SHA-1 — obsoleto, pero presente en infraestructura

SHA-1 (1995, FIPS 180-1) produce un digest de 160 bits (40 hex). Merkle-Damgard con bloques de 512 bits, 80 rondas, estado de 5 palabras de 32 bits.

  • 2005: Wang et al. reducen la complejidad de colision a 269 (teorico).
  • 2017: Google & CWI demuestran colision real — SHAttered — dos PDFs distintos con el mismo SHA-1.
  • NIST SP 800-131A: prohibido para firmas digitales desde 2013.

Detectar SHA-1 en configuraciones TLS (p.ej. TLS_RSA_WITH_AES_128_CBC_SHA) es una señal de configuracion debil. Los certificados SHA-1 fueron eliminados por navegadores en 2017. Git usa SHA-1 para identificar commits (migracion a SHA-256 en curso).

SHA-1 en cipher suite TLS (indicador de debilidad): TLS_RSA_WITH_AES_128_CBC_SHA ← SHA-1 como HMAC TLS_RSA_WITH_AES_256_CBC_SHA ← SHA-1 como HMAC TLS_ECDHE_RSA_WITH_AES_256_CBC_SHA384 ← SHA-384 (OK)
05 / 08

SHA-256 — el estandar operativo actual

SHA-256 (SHA-2, FIPS 180-4) produce un digest de 256 bits (64 hex). Merkle-Damgard con bloques de 512 bits, 64 rondas, ocho palabras de 32 bits en el estado. Las constantes K[0..63] se derivan de las raices cubicas de los primeros 64 numeros primos.

En cada ronda se aplican las operaciones:

  • Ch(e,f,g) — función de elección: \(Ch(e,f,g) = (e \wedge f) \oplus (\lnot e \wedge g)\)
  • Maj(a,b,c) — mayoría: \(Maj(a,b,c) = (a \wedge b) \oplus (a \wedge c) \oplus (b \wedge c)\)
  • \(\Sigma_0(a) = ROTR^{2}(a) \oplus ROTR^{13}(a) \oplus ROTR^{22}(a)\) — suma mayor (mezcla a)
  • \(\Sigma_1(e) = ROTR^{6}(e) \oplus ROTR^{11}(e) \oplus ROTR^{25}(e)\) — suma menor (mezcla e)
  • \(\sigma_0(w) = ROTR^{7}(w) \oplus ROTR^{18}(w) \oplus SHR^{3}(w)\) — expansión del message schedule
SHA-256 en uso real: TLS 1.3 cipher suite: TLS_AES_256_GCM_SHA384 JWT header: {"alg":"HS256"} Bitcoin mining: SHA-256(SHA-256(block_header)) Malware IOC (TI): sha256: 3e23e8160d2b... DFIR cadena custodia: sha256sum evidence.dd > evidence.sha256
06 / 08

SHA-3 / Keccak — construccion esponja

SHA-3 (FIPS 202, 2015) es el ganador del concurso NIST SHA-3 Competition (2007-2012). A diferencia de MD5/SHA-1/SHA-2, no usa la construccion Merkle-Damgard, sino una construccion esponja (sponge function) sobre un estado de 1600 bits.

Estado Keccak: 5×5 matriz de lanes de 64 bits (1600 bits total) FASE ABSORCION: Mensaje (en bloques de rate=136 bytes para SHA3-256) → XOR en los primeros 136 bytes del estado → Aplicar permutacion Keccak-f (24 rondas: θ ρ π χ ι) → Repetir para cada bloque FASE EXPRIMIDO (SQUEEZE): Extraer los primeros 256 bits del estado → digest

Ventajas sobre SHA-2: resistente a length extension attacks por diseño, y ofrece variantes como SHAKE128/SHAKE256 (XOF: salida de longitud variable) y cSHAKE para derivacion de claves customizada.

07 / 08

HMAC — autenticacion con hash

HMAC (Hash-based Message Authentication Code, RFC 2104) combina una clave secreta con una funcion hash para producir un MAC que autentica tanto el origen como la integridad del mensaje.

$$HMAC(K, M) = H\!\Bigl((K \oplus opad) \;\Big\|\; H\!\bigl((K \oplus ipad) \;\|\; M\bigr)\Bigr)$$

ipad = 0x36 repetido blocksize veces opad = 0x5C repetido blocksize veces 1. Si len(K) > blocksize : K = H(K) 2. Si len(K) < blocksize : K = K ‖ 0x00... 3. inner = H( (K ⊕ ipad) ‖ M ) 4. HMAC = H( (K ⊕ opad) ‖ inner )

Uso operativo:

  • JWT (HS256): firma de tokens con HMAC-SHA256.
  • TLS 1.2 PRF: derivacion de claves de sesion.
  • HKDF: TLS 1.3 usa HMAC como base del KDF de dos fases (Extract+Expand).
  • API authentication: AWS SigV4, webhooks GitHub usan HMAC-SHA256.
08 / 08

Hashing en ciberseguridad operativa

El hashing no es solo teoria — aparece constantemente en el trabajo de un analista SOC:

  • Threat Intelligence / IOCs: los Indicators of Compromise son hashes MD5/SHA-1/SHA-256 de muestras de malware. Plataformas como VirusTotal, MISP y OpenCTI indexan por hash.
  • DFIR — cadena de custodia: se calcula el SHA-256 de cada evidencia digital antes y despues del analisis para demostrar integridad. Herramientas: FTK, Autopsy, sha256sum.
  • SIEM/EDR: reglas de deteccion basadas en hash de procesos, DLLs y scripts. Sysmon Event ID 1 incluye el hash SHA-256 del proceso lanzado.
  • TLS cipher suites: el sufijo de la suite indica el hash del HMAC (SHA256, SHA384). TLS 1.3 usa HKDF sobre SHA-256 o SHA-384 segun la suite.
  • Password storage: bcrypt, Argon2, PBKDF2 son KDFs basados en hash con factor de coste. Un hash plano SHA-256 de password es vulnerable a rainbow tables.
