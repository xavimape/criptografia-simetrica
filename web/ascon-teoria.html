¿Qué es ASCON-128a?

┌─────────────────────────────────────────────────────────┐ │ ASCON-128a — Authenticated Encryption with │ │ Associated Data (AEAD) │ ├─────────────────────────────────────────────────────────┤ │ Entradas : Key (128 bits) + Nonce (128 bits) │ │ + Associated Data (variable, no cifrado) │ │ + Plaintext (variable) │ │ │ │ Salidas : Ciphertext (= |Plaintext|) │ │ + Tag de autenticación (128 bits) │ └─────────────────────────────────────────────────────────┘
AEAD: Cifra el plaintext Y autentica tanto el ciphertext como los datos asociados (metadatos visibles pero intocables).
Ganador CAESAR 2019: Seleccionado entre 57 candidatos como el algoritmo de cifrado autenticado recomendado para entornos con restricciones (IoT, sensores, embebidos).
NIST Lightweight Crypto: Finalista del proceso de estandarización NIST para criptografía ligera. Diseñado en IAIK (TU Graz, Austria).
Sin padding costoso: Internamente usa una permutación sponge eficiente — baja latencia, bajo consumo de energía.
02 / 08

Estado interno: 5 × 64 bits

Estado = [ x0 | x1 | x2 | x3 | x4 ] → 320 bits totales Inicialización: x0 = IV = 0x80400c0600000000 (parámetros del algoritmo) x1 = Key_high (bits 127–64 de la clave) x2 = Key_low (bits 63– 0 de la clave) x3 = Nonce_high x4 = Nonce_low IV descodificada: 0x80 = tamaño de clave (128 bits) 0x40 = tasa (rate) = 128 bits por bloque 0x0c = rondas pa = 12 0x06 = rondas pb = 6
Capacidad: Los 192 bits inferiores del estado (x2–x4 parcial) actúan como capacidad y nunca se exponen directamente — garantizan seguridad.
Rate (tasa): Los 128 bits superiores del estado (x0–x1) son el "rate" — la parte que interactúa con los datos en cada paso de absorción/extracción.
Construcción Sponge: ASCON es una esponja dúplex: absorbe datos y exprime resultados usando la misma permutación, alternando según la fase.
03 / 08

S-box de 5 bits

Entrada: una columna vertical de 5 bits (1 bit de cada x0–x4) Operación (64 columnas en paralelo): Paso 1 — Pre-mezcla: x0 ^= x4; x4 ^= x3; x2 ^= x1 Paso 2 — AND-NOT (5 operaciones): t0 = x0 & ~x1 t1 = x1 & ~x2 t2 = x2 & ~x3 t3 = x3 & ~x4 t4 = x4 & ~x0 Paso 3 — XOR final: x0 ^= t1; x1 ^= t2; x2 ^= t3 x3 ^= t4; x4 ^= t0 Paso 4 — Post-mezcla: x1 ^= x0; x0 ^= x4; x3 ^= x2; x2 = ~x2
Sin tablas: Usa solo AND, NOT y XOR → implementable en hardware con pocos transistores y en software sin acceso a memoria (resistente a timing attacks).
No linealidad: Las operaciones AND introducen no-linealidad, haciendo imposible describir la S-box con ecuaciones lineales.
64 en paralelo: Como opera bit a bit sobre los 5 registros, se aplica simultáneamente a las 64 posiciones de bit — paralelismo masivo.

Capa lineal — Difusión

$$\begin{aligned} x_0 &\leftarrow x_0 \oplus \mathrm{ROL}(x_0,\,19) \oplus \mathrm{ROL}(x_0,\,28) \\ x_1 &\leftarrow x_1 \oplus \mathrm{ROL}(x_1,\,61) \oplus \mathrm{ROL}(x_1,\,39) \\ x_2 &\leftarrow x_2 \oplus \mathrm{ROL}(x_2,\,1) \oplus \mathrm{ROL}(x_2,\,6) \\ x_3 &\leftarrow x_3 \oplus \mathrm{ROL}(x_3,\,10) \oplus \mathrm{ROL}(x_3,\,17) \\ x_4 &\leftarrow x_4 \oplus \mathrm{ROL}(x_4,\,7) \oplus \mathrm{ROL}(x_4,\,41) \end{aligned}$$

Capa lineal — rotaciones por registro (ROL = rotate left de 64 bits): Cada xi se mezcla con 2 copias rotadas de sí mismo. Las constantes son distintas por registro → sin simetría explotable.
Difusión intra-registro: Cada bit de un registro afecta a muchos otros bits del mismo registro tras la rotación y XOR.
Difusión inter-registro: La S-box ya creó dependencias entre registros; la capa lineal amplía esas dependencias dentro de cada uno.
MDS implícito: La combinación S-box + capa lineal alcanza rápidamente full diffusion — cualquier cambio en 1 bit afecta el estado completo en pocas rondas.
05 / 08

Permutaciones pa y pb

Una ronda ASCON = Constante RC ⊕ x2 → S-box → Capa lineal Constantes de ronda RC (aplicadas a x2): r0=0xf0 r1=0xe1 r2=0xd2 r3=0xc3 r4=0xb4 r5=0xa5 r6=0x96 r7=0x87 r8=0x78 r9=0x69 r10=0x5a r11=0x4b pa = 12 rondas (índices 0–11) — máxima seguridad pb = 6 rondas (índices 6–11) — velocidad en datos Uso: pa → Inicialización, Finalización pa → Cifrado (ASCON-128a usa pa para mayor seguridad) pb → Absorción de AD
Diferencia ASCON-128 vs -128a: ASCON-128 usa pb (6R) en cifrado con bloques de 64 bits (rate=64). ASCON-128a usa pa (12R) con bloques de 128 bits (rate=128) — el doble de throughput al procesar mensajes largos.
RC evitan simetría: Las constantes de ronda distintas hacen que cada ronda sea única — sin puntos fijos ni simetría explotable en la permutación.
06 / 08

AEAD — Datos Asociados

Associated Data (AD) = datos autenticados pero NO cifrados Ejemplos de AD: • Cabeceras de paquetes de red (IP, puerto, protocolo) • Metadatos de mensajes (timestamp, ID de sesión) • Identificadores de dispositivo en IoT Proceso de absorción: Para cada bloque de 128 bits de AD: 1. Aplicar padding ASCON (bit '1' + ceros) 2. Estado[x0] ⊕= bloque_AD 3. Aplicar pb (6 rondas) Al final: Estado[x4] ⊕= 1 (domain separation)
¿Por qué AD? El receptor puede verificar que los metadatos no fueron alterados, sin necesidad de cifrarlos (eficiencia).
Domain separation: El XOR final (x4 ⊕ 1) garantiza que la fase de AD y la de cifrado produzcan estados internos distintos aunque los datos sean iguales.
AD vacío: Si no hay AD, igual se aplica el domain separation. El estado avanza directamente a la fase de cifrado.
07 / 08

Las 4 fases del algoritmo

┌──────────────────────────────────────────────────────────┐ │ 1. INICIALIZACIÓN │ │ Estado = IV ‖ Key ‖ Nonce │ │ → pa(12 rondas) │ │ → x3 ⊕= Key_high ; x4 ⊕= Key_low │ ├──────────────────────────────────────────────────────────┤ │ 2. ABSORCIÓN AD │ │ Para cada bloque AD de 128 bits: │ │ x0‖x1 ⊕= bloque_AD_padded → pb(6 rondas) │ │ x4 ⊕= 1 (domain separation) │ ├──────────────────────────────────────────────────────────┤ │ 3. CIFRADO │ │ Para cada bloque PT de 128 bits: │ │ CT = x0‖x1 ⊕ PT (extracción) │ │ x0‖x1 = CT (actualizar estado) │ │ → pa(12 rondas) │ ├──────────────────────────────────────────────────────────┤ │ 4. FINALIZACIÓN │ │ x1 ⊕= Key_high ; x2 ⊕= Key_low │ │ → pa(12 rondas) │ │ Tag = (x3 ⊕ Key_high) ‖ (x4 ⊕ Key_low) │ └──────────────────────────────────────────────────────────┘
Secuencial: Cada fase depende del estado producido por la anterior. No se puede paralelizar entre fases.
Key al inicio y al final: La clave aparece en la inicialización y en la finalización — esto liga el tag a la clave, imposibilitando la falsificación sin conocerla.

Tag de autenticación y seguridad

Tag (128 bits) = (x3 ⊕ Key_high) ‖ (x4 ⊕ Key_low) Para descifrar y verificar: 1. Recalcular todo el proceso con Key, Nonce, AD, CT 2. Comparar tag calculado con tag recibido 3. ¿Iguales? → mensaje auténtico e íntegro → aceptar PT ¿Distintos? → mensaje alterado o clave errónea → rechazar Propiedades de seguridad: • Confidencialidad : IND-CCA2 (128 bits de seguridad) • Integridad/Auth : INT-CTXT (128 bits de seguridad) • Seguridad nonce : Nonce debe ser único por (Key, mensaje)
Reutilización de nonce (nonce misuse): Si se usa el mismo nonce dos veces con la misma clave, el atacante puede recuperar el plaintext haciendo XOR de los dos ciphertexts. El nonce DEBE ser único.
Comparación de tags: Siempre compará tags en tiempo constante (evitando timing attacks). Nunca uses comparación byte-a-byte con early exit.
Uso en producción: Para aplicaciones reales usá una librería auditada (libsodium, Bouncy Castle, etc.). Esta implementación es educativa.
