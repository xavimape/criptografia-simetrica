Recursos ChaCha20-Poly1305

Estandares, implementaciones de referencia y recursos para profundizar en el cifrado mas usado en comunicaciones modernas.

Estandares oficiales

RFC

RFC 8439 — ChaCha20 and Poly1305 for IETF Protocols

El estandar definitivo. Incluye descripciones formales, vectores de prueba completos y consideraciones de seguridad. Sustituye al RFC 7539.

rfc-editor.org/rfc/rfc8439 →
Paper

ChaCha, a variant of Salsa20 — D.J. Bernstein (2008)

El paper original de Bernstein donde presenta ChaCha como mejora de Salsa20, con analisis de difusion y justificacion de las constantes de rotacion.

cr.yp.to/chacha →
Paper

The Poly1305-AES Message-Authentication Code — Bernstein (2005)

Diseno original de Poly1305 (version AES). El RFC 8439 usa Poly1305 puro (sin AES), con la clave derivada del keystream de ChaCha20.

cr.yp.to/mac/poly1305 →

Implementaciones de referencia

Python

pyca/cryptography — ChaCha20Poly1305

La libreria criptografica mas usada en Python. Implementa ChaCha20-Poly1305 sobre libsodium / OpenSSL. Uso recomendado en produccion.

cryptography.io/chacha20poly1305 →
C / C++

libsodium — crypto_aead_chacha20poly1305

La implementacion C de referencia. Optimizada con intrinsics NEON y AVX2. La base de muchos protocolos modernos (WireGuard, Signal, minisign).

doc.libsodium.org/chacha20-poly1305 →
Rust

chacha20poly1305 crate — RustCrypto

Implementacion pura en Rust, sin unsafe code, con soporte para ChaCha20Poly1305 y XChaCha20Poly1305 (nonce de 192 bits). Compatible con no-std.

docs.rs/chacha20poly1305 →
Go

golang.org/x/crypto/chacha20poly1305

La implementacion oficial de Go. Incluye XChaCha20-Poly1305 (nonce extendido), usada por Cloudflare y otros en produccion.

pkg.go.dev/chacha20poly1305 →

Donde se usa en la practica

VPN

WireGuard — ChaCha20-Poly1305 como unico cipher

WireGuard usa ChaCha20-Poly1305 exclusivamente (sin negociacion). La decision elimina toda la superficie de ataque de la negociacion de cifrado de OpenVPN / IPSec.

wireguard.com/protocol →
TLS 1.3

RFC 8446 — TLS_CHACHA20_POLY1305_SHA256

TLS 1.3 incluye TLS_CHACHA20_POLY1305_SHA256 como cipher suite mandatorio. Todos los navegadores modernos lo priorizan en dispositivos sin AES-NI.

rfc-editor.org/rfc/rfc8446 →
Mensajeria

Signal Protocol — Double Ratchet + ChaCha20

Signal usa ChaCha20-Poly1305 para cifrar mensajes individuales dentro del Double Ratchet. WhatsApp, Messenger y muchos otros implementan el mismo protocolo.

signal.org/docs/doubleratchet →
SSH

OpenSSH — chacha20-poly1305@openssh.com

OpenSSH implementa ChaCha20-Poly1305 desde la version 6.5 (2014). Es el cipher por defecto en muchas distribuciones Linux por su resistencia a padding oracle attacks.

OpenSSH PROTOCOL.chacha20poly1305 →

Herramientas de analisis y aprendizaje

Online

CyberChef — ChaCha / Poly1305

Herramienta online de GCHQ. Permite aplicar ChaCha20 y Poly1305 paso a paso con entradas hex, ideal para verificar implementaciones.

gchq.github.io/CyberChef →
CLI

OpenSSL 3.x — enc -chacha20

OpenSSL soporta ChaCha20-Poly1305 desde la version 1.1.0. Permite cifrar archivos desde la linea de comandos y verificar vectores del RFC.

openssl.org/docs →
